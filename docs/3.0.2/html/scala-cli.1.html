Scala Commands

NAME

scala-cli - Command-line tool to interact with the Scala language

SYNOPSIS

scala-cli [command] [options]

DESCRIPTION

Scala CLI combines all the features you need to learn and use Scala in your scripts, playgrounds and (single-module) projects.

scala-cli accepts Scala scripts as files that end in .sc. Unlike .scala files, in scripts, any kind of statement is accepted at the top-level:

// hello.sc
val message = "Hello from Scala script"
println(message)

scala-cli and Ammonite are similar, but differ significantly when the user’s code is split in multiple scripts:

  • In Ammonite, a script needs to use import $file directives to use values defined in another script.
  • With scala-cli, all scripts passed can reference each other without such directives.

On the other hand:

  • You can pass a single “entry point” script as input to Ammonite, and Ammonite finds the scripts it depends on via the import $file directives.
  • scala-cli requires all scripts to be passed beforehand, either one-by-one, or by putting them in a directory, and passing the directory to scala-cli.

COMMANDS

Run scala-cli command --help to read about a specific subcommand.
Run scala-cli command --help-full to see full help.

Main Commands

clean
Clean the workspace.
compile
Compile Scala code.
console, repl
Fire-up a Scala REPL.
package
Compile and package Scala code.
  • By default, the package command generates a lightweight JAR that contains only the user’s bytecode.
  • Passing –assembly to the package command generates fat JARs (so-called assemblies).
run
Compile and run Scala code.
test
Compile and test Scala code.

Miscellaneous Commands

about
Print details about this application.
fmt, format, scalafmt
Format Scala code. See scalafmt(1).
browse, metabrowse
Browse Scala code and its dependencies in the browser.
version
Print scala-cli version.

Other Commands

bsp
Start the BSP server.
export
Export current project to Sbt or Mill.
install completions, install-completions
Installs completions into your shell.
setup-ide
Generate a BSP file that you can import into your IDE.
shebang
This command is an equivalent of run, but it changes the way how.
update
Update scala-cli - it works only for installation script.

OPTIONS

Help Options

--usage
Print usage and exit.
-h, -help, --help
Print help message and exit.
--help-full, --full-help
Print help message, including hidden options, and exit.

Scala options

--js
Enable Scala.js. To show more options for Scala.Js pass --help-js.
--native
Enable Scala Native.
-P, --plugin, --compiler-plugin string
Add compiler plugin dependencies
-O, --scala-opt, --scalac-option option
Add a scalac option. Note that options starting with -g, -language, -opt, -P, -target, -V, -W, -X, and -Y are assumed to be Scala compiler options and don’t require to be passed after -O or --scalac-option.
-S, --scala, --scala-version version
Set the Scala version.

Java options

--java-home path
Set the Java home directory.
-j, --jvm jvm-name
Use a specific JVM, such as 14, adopt:11, or graalvm:21, or system.

Note: JVMs are managed by Coursier and are based on the index of jabba(1).

--jar, –jars, --extra-jar, --extra-jars paths
Add extra JARs in the class path.
--compile-only-jar, --compile-only-jars, --extra-compile-only-jar, --extra-compile-only-jars paths
Add extra JARs in the class path, during compilation only.
--source-jar, --source-jars, --extra-source-jar, --extra-source-jars paths
Add extra source JARs.
--resource-dir, --resource-dirs path
Add a resource directory.
-J, --java-opt java-options
Set Java options, such as -Xmx1g.
--java-prop property=value|property
Set Java properties.

Logging Options

-v, --verbose
Increase verbosity (can be specified multiple times).
-q, --quiet
Decrease verbosity.
--progress
Use progress bars.

Entrypoint Options

-M, --main-class main-class
Specify which main class to run.

Scala.JS options

--js-version string
Set the Scala.js version.
--js-mode string
Set the Scala.js mode.

Choices: dev, release.

--js-module-kind [string]
Set the Scala.js module kind.

Choices: commonjs/common, esmodule/es, nomodule/none.

--js-check-ir
 
--js-emit-source-maps
Emit source maps.
--js-dom
Enable jsdom.

Benchmarking options

--jmh
Run JMH benchmarks.
--jmh-version version
Set JMH version.

Scala Native options

--native-version [string]
Set the Scala Native version.
--native-mode [string]
Set Scala Native compilation mode.
--native-gc [string]
Set the Scala Native garbage collector.
--native-clang [string...]
Set the path to the Clang command.
--native-clangpp [string...]
Set the path to the Clang++ command.
--native-linking [string...]
Set extra options passed to clang verbatim during linking.
--native-compile [string...]
List of compile options.

Dependency Options

-d, --dep, --dependency string ...
Add dependencies.
-r, --repo, --repository string ...
Add repositories.

Other options

–home, --home-directory [string]
 
-w, --watch
Watch source files for changes.

EXIT STATUS
