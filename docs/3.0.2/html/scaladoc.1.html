NAME

scaladoc - generate HTML pages of API documentation from Scala 3 source

SYNOPSIS

scaladoc options source_files

DESCRIPTION

The scaladoc command reads class and object definitions, written in the Scala programming language, and generates their API as HTML files.

By default, the generator puts each HTML file in the same directory as its source file. You can specify a separate destination directory with -d (see OPTIONS, below).

The recognised format of comments in source is described in the online documentation.

Supported comment tags are @tparam, @param, @return, @throws, @see, @example, @note, @author, @since, @version.

OPTIONS

The scaladoc command supports the same options as scalac(1) and adds a few more options.

Compiler Options

compiler-option
Any compiler option. See scalac(1).

Doclet Options

-author boolean
Adding authors in docstring with @author Name Surname by default won’t be included in generated HTML documentation. If you would like to label classes with authors explicitly, run scaladoc with this flag.

Default: false.

Similar to option -author of command javadoc.

-comment-syntax syntax
The styling language used for parsing comments.

Choices: markdown or wiki.
Default: markdown.

-doc-canonical-base-url url
A base URL to use as prefix and add canonical URLs to all pages. The canonical URL may be used by search engines to choose the URL that you want people to see in search results. If unset no canonical URLs are generated.
-doc-root-content filename
The file from which the root package documentation should be imported.
-external-mappings:file_pattern::url, ..
Mapping between regexes matching classpath entries and external documentation.

Example: -external-mappings:.*scala.*::scaladoc3::https://scala-lang.org/api/3.x/,.*java.*::javadoc::https://docs.oracle.com/javase/8/docs/api/.
A mapping is of the form regex::[scaladoc|scaladoc|javadoc]::path. You can supply several mappings, separated by commas, as shown in the example.

-groups boolean
Group similar functions together (based on the @group annotation)

Default: false.

-no-link-warnings boolean
Avoid warnings for ambiguous and incorrect links in members look up. Doesn’t affect warnings for incorrect links of assets etc.

Default: false.

-private boolean
Show all types and members. Unless specified, show only public and protected types and members.

Default: false.

Similar to option -private of command javadoc.

-project, -doc-title string
The name of the project.

Aliases with -doc-title to provide compatibility with Scala 2.

-project-footer, -doc-footer string
The string message that appears in a footer section.

Aliases with -doc-footer to provide compatibility with Scala 2.

-project-logo, -doc-logo path
The logo of your project that appears in a top left corner (in /images/ directory).

Aliases with -doc-logo to provide compatibility with Scala 2.

-project-version, -doc-version version
The current version of your project.

Default: "".

Similar to option -version of command javadoc.

-revision revision
Revision (branch or ref) used to build project. Useful with sourcelinks to prevent them from pointing always to the newest master that is subject to changes.

Default: "".

-siteroot directory
A directory containing static files from which to generate documentation.

Default: ./docs/.

-skip-by-id string{,string}
Identifiers of packages or top-level classes to skip when generating documentation.
-skip-by-regex string{,string}
Regexes that match fully qualified names of packages or top-level classes to skip when generating documentation.
-skip-packages string{,string} (deprecated)
Use -skip-by-id or -skip-by-regex instead.
-snippet-compiler string{,string}
Snippet compiler arguments provide a way to configure snippet type checking.

Usage example: -snippet-compiler:my/path/nc=nocompile,my/path/f=fail,compile.

-social-links string{,string}
Links to social sites. [github|twitter|gitter|discord]::link syntax is used.

custom::link::white_icon_name::black_icon_name is also allowed, in this case icons must be present in the images/ directory.

-source-links string{,string}
Source links provide a mapping between file in documentation and code repository.
-versions-dictionary-url url
A URL pointing to a JSON document containing a dictionary version label -> documentation location. The JSON file has single property “versions” that holds dictionary of labels of specific docs and URL pointing to their index.html top-level file. Useful for libraries that maintain different versions of their documentation.

Default: "".

Advanced Options

-Xwiki-syntax
Retains the Scala 2 behavior of using Wiki syntax in scaladoc.

See document Scaladoc settings compatibility between Scala 2 and Scala 3 for more information.

Private Options

-Ydocument-synthetic-types boolean
Include pages providing documentation for the intrinsic types (e.g. Any, Nothing) to the docs. The setting is useful only for stdlib because scaladoc for Scala 3 relies on TASTy files, but we cannot provide them for intrinsic types since they are embedded in the compiler. All other users should not concern with this setting.

Default: false.

-Ygenerate-inkuire boolean
Generate InkuireDB and enables Hoogle-like searches.

Default: false.

Note:  Inkuire works as a JavaScript worker in the browser thanks to the power of Scala.js.

-Ysnippet-compiler-debug
Setting this option makes snippet compiler print the snippet as it is compiled (after wrapping).

EXAMPLES

EXIT STATUS

scaladoc returns a zero exit status if it succeeds at processing the specified input files. Non zero is returned in case of failure.
