NAME

sbt - Interactive build tool for Scala, Java, and more.

SYNOPSIS

sbt [options] [file...]

DESCRIPTION

sbt is built for Scala and Java projects. One of the examples of Scala-specific feature is the ability to cross build your project against multiple Scala versions.

build.sbt is a Scala-based DSL to express parallel processing task graph. Typos in build.sbt will be caught as a compilation error.

With Zinc incremental compiler and file watch (~), edit-compile-test loop is fast and incremental.

Adding support for new tasks and platforms (like Scala.js) is as easy as writing build.sbt.

Further features include :

  • Library management support using Coursier.
  • Continuous compilation and testing with triggered execution.
  • Supports testing with ScalaCheck, specs2, and ScalaTest. JUnit is supported by a plugin.
  • Starts the Scala REPL with project classes and dependencies on the classpath.
  • Modularization supported with sub-projects.
  • External project support (list a git repository as a dependency!)
  • Parallel task execution, including parallel test execution.

OPTIONS

-Dproperty=value
Pass -Dproperty=value directly to the Java runtime.
-h, --help
Print this message.
-v, --verbose
This runner is chattier.
-V, --version
Print sbt version information.
--numeric-version
Print the numeric sbt version (sbt sbtVersion).
--script-version
Print the version of sbt script.
-d, --debug
Set sbt log level to debug.
-debug-inc, --debug-inc
Enable extra debugging for the incremental debugger.
--no-colors
Disable ANSI color codes.
--color=auto|always|true|false|never
Enable or disable ANSI color codes (sbt 1.3 and above).
--supershell=auto|always|true|false|never
Enable or disable supershell (sbt 1.3 and above).
--traces
Generate Trace Event report on shutdown (sbt 1.3 and above).
--timings
Display task timings report on shutdown.
--sbt-create
Start sbt command even if current directory contains no sbt project.
--sbt-dir path
Set path to global settings/plugins directory.

Default (Unix): ~/.sbt
Default (Windows): %USERPROFILE%\.sbt

--sbt-boot path
Set path to shared boot directory.

Default: ~/.sbt/boot in 0.11 series.

--sbt-cache path
Set path to global cache directory.

Default: operating system specific.

--ivy path
Set path to local Ivy repository.

Default: ~/.ivy2/

--mem integer
Set memory size.

Default: 1024 bytes.

--no-share
Use all local caches; no sharing.
--no-global
Use global caches, but does not use global ~/.sbt directory.
--jvm-debug port
Enable on JVM debugging, open at the given port.
--sbt-version version
Use the specified version of sbt.

Default: File project/build.properties if present, otherwise latest sbt release.

--sbt-jar path
Use the specified jar as the sbt launcher.
--java-home path
Specify an alternate JAVA_HOME.

Default: java command from PATH.

ENVIRONMENT

Environment Variables

JAVA_HOME
Specify JDK/JRE home directory. This directory is used to locate the java command.
JAVA_OPTS
If unset uses -Dfile.encoding=UTF-8.
SBT_OPTS
If unset uses "".

Note: SBT_OPTS acts as a superset of JAVA_OPTS allowing -X* and -Dproperty=value as well as Sbt script specific features such as -v and --supershell=false.

System Properties

The following System properties influence Sbt execution :

sbt.banner boolean
Show a welcome banner advertising new features.

Default: true (in 1.3 and above).

sbt.boot.directory directory
Set path to shared boot directory. You should define sbt.boot.directory explicitly if you are using sbt in a CI environment and want to cache sbt boot classes between runs.

Default (Unix): ~/.sbt/boot/
Default (Windows): %USERPROFILE%\.sbt\boot\.

sbt.coursier boolean
Use Coursier to retrieve packages (see also sbt.ivy).

Default: true (in 1.3 and above).

sbt.coursier.home directory
Set location of coursier artifact cache, where the default is defined by Coursier cache resolution logic. You can verify the value with the command csrCacheDirectory. You should define sbt.coursier.home explicitly if you are using coursier in a CI environment and want to cache artifacts between runs.

Default: CoursierDependencyResolution.defaultCacheLocation (in 1.3 and above).

sbt.ivy boolean
Use Ivy to retrieve packages.

Default: true (in <1.3).

sbt.ivy.home directory
Set the directory containing the local Ivy repository and artifact cache. You should define sbt.ivy.home explicitly if you are using sbt in a CI environment and want to cache ivy artifacts between runs.

Default (Unix): ~/.ivy2/
Default (Windows): %USERPROFILE%\.ivy2\

sbt.log.noformat boolean
If true, disable ANSI color codes. Useful on build servers or terminals that do not support color.

Default: false.

sbt.offline boolean
Avoid retrieving classes from repositories.

Default: false.

sbt.supershell boolean
Use supershell (show progress at bottom of shell). true if color is enabled.
sbt.version version
Set sbt version to use, usually taken from file project/build.properties.

Configuration Files

project/build.properties
If not present sbt creates file project/build.properties with the following initial contents :

sbt.version=1.6.2

.jvmopts
If file .jvmopts exists in the current directory, its contents are appended to JAVA_OPTS.
.sbtopts
If file .sbtopts exists in the current directory, its contents are prepended to the runner arguments.
$SBT_HOME/conf/sbtconfig.txt
If this file exists, it is prepended to the runner arguments.

Note: In the case of duplicated or conflicting options, the order above shows precedence: JAVA_OPTS lowest, command line options highest.

EXIT STATUS

The sbt command returns a zero exit status if it succeeds. Non zero is returned in case of any error.
