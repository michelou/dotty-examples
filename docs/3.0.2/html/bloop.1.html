NAME

bloop - Build server that runs in the backgroud and serves build requests

SYNOPSIS

bloop [options] [command [command-options]]

DESCRIPTION

Bloop is a build server and CLI tool for the Scala programming language developed by the Scala Center.

Bloop has two main goals:

  • Compiles, tests and runs Scala code as fast as possible.
  • Integrates easily with build tools, command-line applications, editors and custom tooling.

Clean Options

--projects, -p, --project value
Set the projects to clean (you can specify multiple). If none, all are cleaned.
--include-dependencies, --propagate
Clean a project and all its dependencies.

Default: false

--cascade
Clean a project and all projects depending on it.

Default: false

--config-dir, -c path
Set the path to the bloop config directory (containing the .bloop file).

Default: .bloop in the current working directory.

--version, -v
If set, print the about section at the beginning of the execution.

Default: false

--verbose
If set, print out debugging information to stderr.

Default: false

--no-color
If set, do not color output.

Default: false

--debug value
Debug the execution of a concrete task.
compile
You can’t run bloop compile or bloop test without specifying a project. There is no implicit root project.

Compile Options

--projects, -p, –project value
Defined the projects to compile (will be inferred from remaining cli args).
--incremental
Compile the project incrementally.

Default: true

--pipeline
Pipeline the compilation of modules in your build.

Default: false

--reporter value
Pick reporter to show compilation messages. By default, bloop’s used.

EXAMPLES

A Bloop server is running on port 8212 per default.

  • To spawn a bloop server on another port run the command
    bloop server nailgun_port.
  • To kill the running server run the command
    bloop ng-stop

Spawn the Bloop server on port 8888 (Windows)

start "bloop:8888" bloop server 8888

EXIT STATUS

bloop returns a zero exit status if it succeeds at processing the specified input files. Non zero is returned in case of failure.
