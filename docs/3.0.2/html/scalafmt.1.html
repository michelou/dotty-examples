NAME

scalafmt - Source code formatter for the Scala language

SYNOPSIS

scalafmt [options] [file...]

DESCRIPTION

file... is a file or directory in which all *.scala files are to be formatted.

If file starts with @, refers to path listing files to be formatted (with @- referring to standard input as a special case).

OPTIONS

-h, --help
Print this usage text.
-v, --version
Print Scalafmt version.
--stdout
Write formatted files to stdout.
--exclude file|directory
File or directory, when absent all *.scala files are formatted.
--respect-project-filters
Use project filters even when specific files to format are provided.
-c <path>, --config path
Specify a configration definad as .scalafmt.conf.
--config-str value
Specify a configuration defined as a string.
--stdin
Read from stdin and print to stdout.
--no-stderr
Don’t use stdrerr for messages, output to stdout.
--assume-filename value
When using --stdin, use --assume-filename to hint to scalafmt that the input is an .sbt file.
--reportError
Exit with status 1 if any mis-formatted code found.
--test
Test for mis-formatted code only, exits with status 1 on failure.
--check
Test for mis-formatted code only, exits with status 1 on first failure.
--diff (Deprecated)
Format files listed in git diff against master.

Use --mode diff instead.

--mode value
Set the files to be formatted fetching mode.

Choices:

  • diff: format files listed in git diff against master.
  • changed: format files listed in git status (latest changes against previous commit).
  • any: format any files found in current directory.
  • anygit: format any git-tracked files found in current directory.
--diff-branch value
If set, only format edited files in git diff against provided branch. Has no effect if mode set to changed.
--build-info
Print build information.
--quiet
Don’t print out stuff to console.
--debug
Print out diagnostics to console.
--non-interactive
Disable fancy progress bar, useful in ci or sbt plugin.
--list
List files that are different from Scalafmt formatting.

ENVIRONMENT

Environment Variables

JAVA_HOME
Specify JDK/JRE home directory. This directory is used to locate the java command.

Configuration Files

Configuration for the scalafmt command is defined in a plain text file .scalafmt.conf using the HOCON syntax.

scalafmt configuration is determined in this order:

  1. .scalafmt.conf file in current directory
  2. .scalafmt.conf inside root directory of current Git repository
  3. no configuration file, use default style.

Here is an example .scalafmt.conf :

align.preset = more  // For pretty alignment.
maxColumn = 100      // For my wide 30" display.

EXAMPLES

Format all files in the current project.

scalafmt

Throw exception on mis-formatted files, won’t write to files.

scalafmt --test

Format all files that were edited in git diff against master branch.

scalafmt --mode diff

Format files listed in git status (latest changes against previous commit).

scalafmt --mode changed

Same as --diff, except against branch 2.x.

scalafmt --diff-branch 2.x

Read from stdin and print to stdout.

scalafmt --stdin

Required when using --stdin to format .sbt files.

scalafmt --stdin --assume-filename foo.sbt < foo.sbt

Write formatted contents to file.

scalafmt Code1.scala A.scala

Print formatted contents to stdout.

scalafmt --stdout Code.scala

Format all files in directory excluding target

scalafmt --exclude target

Read custom style from file.

scalafmt --config .scalafmt.conf

Define custom style as a flag, must be quoted:

scalafmt --config-str "style=IntelliJ"

EXIT STATUS

The scalafmt command returns a zero exit status if it succeeds. Non zero is returned in case of any error.
