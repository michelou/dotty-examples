NAME

jabba - Java Version Manager

SYNOPSIS

jabba [jabba-options] [command [command-options]]

DESCRIPTION

jabba is written in Go and is inspired by nvm (Node.js).

jabba keeps everything under ~/.jabba/ on Linux/MacOSX resp. %USERPROFILE%\.jabba\ on Windows. If at any point of time you decide to uninstall jabba just remove this directory.

COMMANDS

ls
List installed versions

OPTIONS

tbd

ENVIRONMENT

.jabbarc
 

EXAMPLES

List the available JDKs.

jabba ls-remote

You can use any valid semver range to narrow down the list.
jabba ls-remote zulu@~1.8.60
jabba ls-remote "*@>=1.6.45 <1.9" --latest=minor

Install the Zulu JDK version 1.11.0-10.

jabba install zulu@1.11.0-10

Install Adopt OpenJDK (Hotspot)

jabba install adopt@1.8-0

Install GraalVM CE version 21.1.0

jabba install graalvm-ce-java11@21.1.0

Install from custom URL (supported qualifiers: zip, tgz, tgx, dmg, bin, exe).

jabba install 1.8.0-custom=tgz+http://example.com/distribution.tar.gz jabba install 1.8.0-custom=tgx+http://example.com/distribution.tar.xz jabba install 1.8.0-custom=zip+file:///opt/distribution.zip

Uninstall the Zulu JDK

jabba uninstall zulu@1.6.77

Link system JDK

jabba link system@1.8.72 /Library/Java/JavaVirtualMachines/jdk1.8.0_72.jdk

List the installed JDKs.

jabba ls

Switch to a different version of JDK (it must be already installed)

jabba use adopt@1.8
jabba use zulu@~1.6.97

Switch to the JDK specified in .jabbarc (since 0.5.0).

echo "1.8" > .jabbarc
jabba use

Set default Java version on shell (since 0.2.0). This version will automatically be “jabba use”d every time you open up a new terminal.

jabba alias default 1.8

EXIT STATUS

The jabba command returns a zero exit status if it succeeds. Non zero is returned in case of any error.
