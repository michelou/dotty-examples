NAME

gradle - Build automation tool for Java, Scala, Android, Kotlin, C/C++, Groovy and more.

SYNOPSIS

gradle [taskName...] [option..]

DESCRIPTION

Gradle is a build tool with a focus on build automation and support for multi-language development. If you are building, testing, publishing, and deploying software on any platform, Gradle offers a flexible model that can support the entire development lifecycle from compiling and packaging code to publishing web sites.

Gradle has been designed to support build automation across multiple languages and platforms including Java, Scala, Android, Kotlin, C/C++, and Groovy, and is closely integrated with development tools and continuous integration servers including Eclipse, IntelliJ, and Jenkins.

COMMANDS

init
Create a new Gradle build; option --type type specifies the project type.

Example: gradle init --type scala-application options

OPTIONS

Debugging Options

-?, -h, --help
Show a help message with all available CLI options.
-v, --version
Print Gradle, Groovy, Ant, JVM, and operating system version information.
-S, --full-stacktrace
Print out the full (very verbose) stacktrace for any exceptions. See also logging options.
-s, --stacktrace
Print out the stacktrace also for user exceptions (e.g. compile error). See also logging options.
--scan
Create a build scan with fine-grained information about all aspects of your Gradle build.
-Dorg.gradle.debug=true
Debug Gradle client (non-Daemon) process. Gradle will wait for you to attach a debugger at localhost:5005 by default.
-Dorg.gradle.debug.port=number
Specify the port number to listen on when debug is enabled.

Default is 5005.

-Dorg.gradle.debug.server=(true,false)
If set to true and debugging is enabled, Gradle will run the build with the socket-attach mode of the debugger. Otherwise, the socket-listen mode is used.

Default: true

-Dorg.gradle.debug.suspend=(true,false)
When set to true and debugging is enabled, the JVM running Gradle will suspend until a debugger is attached.

Default: true

-Dorg.gradle.daemon.debug=true
Debug Gradle Daemon process.

Performance Options

--build-cache, --no-build-cache
Toggles the Gradle build cache. Gradle will try to reuse outputs from previous builds.

Default: off.

--configuration-cache, --no-configuration-cache
Toggle the Configuration Cache. Gradle will try to reuse the build configuration from previous builds.

Default: off.

--configuration-cache-problems=(fail,warn)
Configures how the configuration cache handles problems.

Default: fail

Set to warn to report problems without failing the build.

Set to fail to report problems and fail the build if there are any problems.

--configure-on-demand, --no-configure-on-demand
Toggle Configure-on-demand. Only relevant projects are configured in this build run.

Default: off.

--max-workers
Set the maximum number of workers that Gradle may use. Default is number of processors.
--parallel, --no-parallel
Build projects in parallel. For limitations of this option, see Parallel Project Execution.

Default: off.

--priority priority
Specify the scheduling priority for the Gradle daemon and all processes launched by it. Values are normal or low.

Default: normal.

--profile
Generate a high-level performance report in the $buildDir/reports/profile directory. --scan is preferred.
--scan
Generate a build scan with detailed performance diagnostics.
--watch-fs, --no-watch-fs
Toggle watching the file system. When enabled Gradle re-uses information it collects about the file system between builds. Enabled by default on operating systems where Gradle supports this feature.

Daemon Options

--daemon, --no-daemon
Use the Gradle Daemon to run the build. Starts the daemon if not running or existing daemon busy.

Default: on

--foreground
Starts the Gradle Daemon in a foreground process.
--status (Standalone command)
Run gradle –status to list running and recently stopped Gradle daemons. Only displays daemons of the same Gradle version.
--stop (Standalone command)
Run gradle –stop to stop all Gradle Daemons of the same version.
-Dorg.gradle.daemon.idletimeout=millis
Gradle Daemon will stop itself after this number of milliseconds of idle time.

Default: 10800000 (3 hours).

Logging Options

-Dorg.gradle.logging.level=level
Set logging level via Gradle properties.

Choices: quiet, warn, lifecycle, info, debug.

-q, --quiet
Log errors only.
-w, --warn
Set log level to warn.
-i, --info
Set log level to info.
-d, --debug
Log in debug mode (includes normal stacktrace).

Execution Options

--include-build
Run the build as a composite, including the specified build. See Composite Builds.
--offline
Specify that the build should operate without accessing network resources. Learn more about options to override dependency caching.
--refresh-dependencies
Refresh the state of dependencies. Learn more about how to use this in the dependency management docs.
-m, --dry-run
Run Gradle with all task actions disabled. Use this to show which task would have executed.
-t, --continuous
Enable continuous build. Gradle does not exit and will re-execute tasks when task file inputs change. See Continuous Build for more details.
--write-locks
Indicate that all resolved configurations that are lockable should have their lock state persisted. Learn more about this in dependency locking.
--update-locks group:name[,...]
Indicate that versions for the specified modules have to be updated in the lock file. This flag also implies --write-locks. Learn more about this in dependency locking.
-a, --no-rebuild
Do not rebuild project dependencies. Useful for debugging and fine-tuning buildSrc, but can lead to wrong results. Use with caution!

Dependency Verification Options

-F, --dependency-verification=mode
Configure the dependency verification mode, see what the options mean here.

Choices: strict (default), lenient, off.

-M, --write-verification-metadata
Generates checksums for dependencies used in the project (comma-separated list) for dependency verification. See how to bootstrap dependency verification.
--refresh-keys
Refresh the public keys used for dependency verification.
--export-keys
Exports the public keys used for dependency verification.

Environment Options

-b, --build-file (deprecated)
Specify the build file.

Example: gradle --build-file=foo.gradle.

Defaults: build.gradle, then build.gradle.kts.

-c, --settings-file=file (deprecated)
Specify the settings file.

For example: gradle --settings-file=somewhere/else/settings.gradle

-g, --gradle-user-home
Specify the Gradle user home directory. The default is the .gradle directory in the user’s home directory.
-p, --project-dir
Specify the start directory for Gradle.

Default: current directory.

--project-cache-dir
Specify the project-specific cache directory. Default value is .gradle in the root project directory.
-D, --system-prop
Set a system property of the JVM, for example -Dmyprop=myvalue. See System Properties.
-I, --init-script
Specifies an initialization script. See Init Scripts.
-P, --project-prop property=value
Set a project property of the root project, for example -Pmyprop=myvalue. See Project Properties.
-Dorg.gradle.jvmargs
Set the JVM arguments.
-Dorg.gradle.java.home=path
Set the JDK home directory.

EXIT STATUS

The gradle command returns a zero exit status if it succeeds. Non zero is returned in case of any error.
