Scala 3 Commands

NAME

scala - Code runner for the Scala 3 language

SYNOPSIS

scala options [class_name|source_file]

DESCRIPTION

The scala command starts a Scala application using a Java runtime environment (see section ENVIRONMENT). The Scala code to run is specified in one of three ways:

  1. With no arguments specified, the Scala REPL starts and reads user commands interactively.
  2. A fully qualified name of a top-level Scala object is specified. The object should previously have been compiled using scalac(1).
  3. A file containing Scala code is specified.

The scala command checks whether a file of the specified name exists. If it does, then it will treat it as a script file; if it does not, then it will treat it as the name of an object.

In all three cases, arbitrary scalac options may be specified. The most common option is to specify a class path with -classpath, but see the scalac(1) page for full details.

If an object is specified to run, then that object must be a top-level Scala object with the specified name. The object must define a method main with the following signature:

def main(args: Array[String]): Unit

The method must return a Unit value, and it must accept a String array as a parameter. All arguments specified on the command line will be passed as arguments to the main method.

If a script file is specified to run, then the file is read and all Scala statements and declarations in the file are processed in order. Any arguments specified will be available via the args variable.

Script files may have an optional header that is ignored if present. There are two ways to format the header:

  • either beginning with #! and ending with !#,
  • or beginning with ::#! and ending with ::!#.

Such a header must have each header boundary start at the beginning of a line. Headers can be used to make stand-alone script files, as shown in the EXAMPLES section below.

OPTIONS

The scala command supports the same options as scalac(1) and adds a few more options.

Standard Options

compiler-option
Any standard compiler option. See scalac(1).
-nosave
 
-save
Save this compiled version of a script in order to speed up its later executions. When running a script, save the compiled version in a file with the same name as the script but with a .jar extension. On subsequent runs of the same script, the pre-compiled .jar file is used if it is newer than the script file.

This is similar to option -savecompiled of Scala 2.

-with-compiler
Extend the class path passed to the Java runtime with the compiler libraries (in the following LIB=$SCALA_HOME/lib):

$LIB/scala3-compiler_3-3.0.2.jar
$LIB/tasty-core_3-3.0.2.jar
$LIB/scala3-interfaces-3.0.2.jar
$LIB/scala-asm-3.0.2-scala-1.jar
$LIB/scala3-staging_3-3.0.2.jar
$LIB/scala3-tasty-inspector_3-3.0.2.jar

Reminder: In normal case the class path is constructed as follows :

  1. If the user specifies option -classpath cpath its value is appended to the base class path, i.e.

$LIB/scala3-library_3-3.0.2.jar:$LIB/scala-library-2.13.6.jar:<cpath>

  1. Otherwise the current directory (“.”) is appended to the base class path, i.e.

$LIB/scala3-library_3-3.0.2.jar:$LIB/scala-library-2.13.6.jar:.

Warning Options

-Wname
Any warning compiler option. See scalac(1).

Advanced Options

-Xname
Any advanced compiler option. See scalac(1).

Private Options

-Yname
Any private compiler option. See scalac(1).

ENVIRONMENT

Environment Variables

JAVACMD
Specify the java command to be used for running the Scala code. Arguments may be specified as part of the environment variable; spaces, quotation marks, etc., will be passed directly to the shell for expansion.
JAVA_HOME
Specify JDK/JRE home directory. This directory is used to locate the java command unless the JAVACMD variable is set.
JAVA_OPTS
Specify the options to be passed to the java command defined by JAVACMD.

Example (Unix): export JAVA_OPTS="-Xmx512M -Xms16M -Xss16M"
Example (Windows): set JAVA_OPTS=-Xmx512M -Xms16M -Xss16M

SCALA_OPTS
Specify the options to be passed to the scala command.

System Properties

The Scala 3 commands have access to the System properties of the underlying Java platform. System properties include information about the current user, the current version of the Java runtime, and the character used to separate components of a file path name.

Furthermore the scala command works together with the following System properties, some of which are specific to Scala 3 :

java.class.path
Path used to find directories and JAR archives containing class files. Elements of the class path are separated by a platform-specific character specified in the path.separator property.
java.ext.dirs
Same as compiler option -javaextdirs.

See Oracle’s guide Extension Mechanism Architecture for more information.

scala.ext.dirs
 
scala.home
Path of the Scala 3 software installation.
scala.usejavacp
 
sun.boot.class.path
 

EXAMPLES

Here are some examples of running Scala code:

Execute a Scala program generated in the current directory

scala hello.HelloWorld

Execute a Scala program generated in the user-defined directory classes

scala -classpath classes hello.HelloWorld

Execute a Scala program using a user-defined java command

env JAVACMD=/usr/local/bin/java scala -classpath classes hello.HelloWorld

Execute a Scala program using JVM options

env JAVACMD=/usr/local/bin/java JAVA_OPTS="-Dmsg=hello -enableassertions" scala -classpath classes hello.HelloWorld
 

Here is a complete Scala script for Unix:

#!/bin/sh
exec scala "$0" "$@"
!#
Console.println("Hello, world!")
args.toList foreach Console.println

Here is a complete Scala script for MS Windows:

::#!
@echo off
call scala %0 %*
goto :eof
::!#
Console.println("Hello, world!")
args.toList foreach Console.println

If you want to use the compilation cache to speed up multiple executions of the script, then add -save to the scala command:

#!/bin/sh
exec scala -save "$0" "$@"
!#
Console.println("Hello, world!")
args.toList foreach Console.println

EXIT STATUS

The scala command returns a zero exit status if it succeeds. Non zero is returned in case of any error. If a script or top-level object is executed and returns a value, then that return value is passed on to scala.
