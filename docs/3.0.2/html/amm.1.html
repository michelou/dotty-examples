NAME

amm - Ammonite-REPL, a Scala REPL and script runner

SYNOPSIS

amm [ammonite-options] [script-file[script-options]]

DESCRIPTION

Command amm is a modernized Scala REPL. It supports syntax highlighting, multi-line editing, loading of Maven artifacts directly in the REPL, and many other quality-of-life improvements missing in the default Scala REPL.

OPTIONS

--no-default-predef
Disable the default predef and run Ammonite with the minimal predef possible.
-s, --silent
Make ivy logs go silent instead of printing though failures will still throw exception.
-w, --watch
Watch and re-run your scripts when they change.
--bsp
Run a BSP server against the passed scripts.
-c, --code string
Pass in code to be run immediately in the REPL.
-h, --home path
The home directory of the Ammonite-REPL (where it looks for config and caches).
-p, --predef path
Lets you load your predef from a custom location, rather than the “default” location in your Ammonite home.
--color boolean
Enable or disable colored output; by default colors are enabled in both REPL and scripts if the console is interactive, and disabled otherwise.
--thin
Hide parts of the core of Ammonite and some of its dependencies. By default, the core of Ammonite and all of its dependencies can be seen by users from the Ammonite session. This option mitigates that via class loader isolation.
--help
Print this message.
-v, --version
Show Ammonite’s version.
--predef-code string
Specify any commands you want to execute at the start of the REPL session.
--no-home-predef
Disable the default behavior of loading predef files from your ~/.ammonite/predef.sc, predefScript.sc, or predefShared.sc. You can choose an additional predef to use using --predef.
-b, --banner string
Customize the welcome banner that gets shown when Ammonite starts.
--no-remote-logging (deprecated)
Disable remote logging of the number of times a REPL starts and runs commands.
--class-based
Wrap user code in classes rather than singletons, typically for Java serialization friendliness.

EXIT STATUS

The amm command returns a zero exit status if it succeeds. Non zero is returned in case of any error.
