NAME

cs - Scala application and artifact manager

SYNOPSIS

cs [<coursier-options>] [<command>] [<command-options>]

DESCRIPTION

Coursier is the Scala application and artifact manager. It can install Scala applications and setup your Scala development environment. It can also download and cache artifacts from the web.

COMMANDS

channel
See Channel Options).
fetch
Download a particular library by providing the Maven coordinates. See Fecth Options).

Example: cs fetch org.scala-lang:scala-library:2.13.8

get
See Get Options).
install
Install an application from its descriptor (see Install Options).
java
See Java Options).
launch
See Launch Options).

Example: cs launch <org:name:version|app-name[:version]*>

resolve
Command resolve only download the metadata files.
update
See Update Options).

EXAMPLES

Install the latest available version of the scalafmt command.

cs install scalafmt

cs install –channel io.get-coursier:apps-contrib proguard

cs install –contrib proguard

Identify the transitive dependencies of package akka-actor:2.6.19 available from the Maven repository:

$ cs resolve com.typesafe.akka::akka-actor:2.6.19
com.typesafe:config:1.4.2:default
com.typesafe.akka:akka-actor_2.13:2.6.19:default
org.scala-lang:scala-library:2.13.8:default
org.scala-lang.modules:scala-java8-compat_2.13:1.0.0:default

Update only Ammonite REPL to the latest available version.

$ cs update amm

EXIT STATUS

The cs command returns a zero exit status if it succeeds. Non zero is returned in case of any error. If a script or top-level object is executed and returns a value, then that return value is passed on to cs.
